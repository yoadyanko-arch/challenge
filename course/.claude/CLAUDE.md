# פרויקט קורס

## פקודות נפוצות (Commands)
* **הפעלה / הרצה בסביבת הפיתוח:** `<כאן באה הפקודה - למשל: npm run dev או python main.py>`
* **הרצת בדיקות (Tests):** `<למשל: npm run test או pytest>`
* **בניית הפרויקט (Build):** `<למשל: npm run build>`
* **בדיקת תקינות קוד (Linting):** `<למשל: npm run lint>`

## הנחיות כתיבת קוד (Style Guidelines)
* **שמות:** להשתמש בשמות משמעותיים למשתנים, פונקציות ומחלקות. (למשל: `camelCase` למשתנים ו-`PascalCase` למחלקות ברוב השפות).
* **סגנון:** העדפה לקוד קריא, מודולרי ותמציתי.
* **הערות:** לכתוב הערות כדי להסביר את ה"למה" (Why) ולא בהכרח את ה"מה" (What), אלא אם הקוד מורכב.
* **שפות וכלים עיקריים:** `<למשל: השתמש ב-TypeScript עם תמיכה מתאימה ב-React>`

## ארכיטקטורת תיקיות (Architecture)
* `src/` - מכיל את קבצי קוד המקור (Source Code).
* `tests/` - מכיל את כל הבדיקות.
* `public/` - קבצים סטטיים.

## הוראות מיוחדות נוספות
* יש לוודא כי כל הוספה של פיצ'ר חדש מלווה בבדיקה (Test) מתאימה.
* אל תיצור קבצים מחוץ לתיקיית הפרויקט אלא אם באמת נדרש לכך.

> **הערה:** אפשר למחוק ולשנות את התבנית הזו לגמרי כך שתתאים בול למה שאתה צריך עבור הקורס או הפרויקט שאתה בונה.
