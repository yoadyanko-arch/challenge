# הנחיות כתיבת קוד (Style Guidelines)

* **שמות:** להשתמש בשמות משמעותיים למשתנים, פונקציות ומחלקות. (למשל: `camelCase` למשתנים ו-`PascalCase` למחלקות ברוב השפות).
* **סגנון:** העדפה לקוד קריא, מודולרי ותמציתי.
* **הערות:** לכתוב הערות כדי להסביר את ה"למה" (Why) ולא בהכרח את ה"מה" (What), אלא אם הקוד מורכב.
* **שפות וכלים עיקריים:** `<למשל: השתמש ב-TypeScript עם תמיכה מתאימה ב-React>`
